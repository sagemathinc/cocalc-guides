# Product Notes

## Positioning choices

- Keep the tone explicitly respectful. The point is to help users map existing
  mental models to CoCalc, not to argue that other products are bad.
- Lead with "center of gravity" instead of feature matrices. A product can be
  excellent and still be organized around a different primitive.
- The strongest one-line positioning remains: CoCalc is a durable collaborative
  computational project workspace, with Jupyter notebooks as a first-class part
  of that project.

## Follow-up comparison guides

- CoCalc as a hosted agent sandbox environment.
- CoCalc for coding, Linux, and learning command-line work in a browser.
- CoCalc for LaTeX and technical writing, especially compared with Overleaf.
- CoCalc for teaching technical courses with notebooks, terminals, nbgrader,
  shared infrastructure, and agent support.

## Product ideas surfaced while writing

- The public feature pages already have a compare page; this guide could become
  a softer, more visual route into that same message.
- A compact "choose by workflow" widget could be useful on the public site:
  quick notebook, editor-first repo, analytics platform, cloud ML platform,
  durable collaborative project.
- The guide would be stronger with a few real screenshots once CoCalc-AI public
  UI screenshots are stable enough to use consistently.
