# Product Notes From Drafting This Guide

These notes are not part of the public guide. They capture product ideas that
became sharper while writing the long-running computation guide.

## Strong Ideas

### Run receipt

Long computations want a first-class run receipt:

- command
- working directory
- start and stop time
- output directory
- log path
- git commit or file hashes
- status: running, failed, partial, complete

This could be generated by Codex or by a project command that wraps a terminal
run.

### Computation supervisor prompt

CoCalc-AI could offer an agent action such as:

> Manage this long-running computation.

The action would create an output directory, tee logs, run a small smoke test,
start the real run, periodically inspect progress, and summarize failures.

### Output inventory

A small product feature could scan a run directory and classify outputs:

- complete
- missing
- failed
- stale
- suspicious size

This is simple, but it gives Codex much better context and gives users a fast
way to understand partial results.

### Terminal session continuity

The guide depends heavily on shared terminal history. It would be useful if a
long-running terminal session could be explicitly named, pinned to a run, and
attached to a Codex thread.

## Smaller Ideas

### Snapshot before risky reruns

If a rerun will modify many outputs or update the root filesystem, offer a fast
snapshot prompt.

### Notebook summary from run directory

Codex could create a standard notebook from a run directory: load outputs,
summarize coverage, plot quick diagnostics, and list failures.

### Restart after project/server interruption

After a project restart, CoCalc-AI could detect recently active long-run
terminals or run receipts and offer to inspect what survived.
