# Python in CoCalc: From Notebook to Script to Paper

This guide positions Python in CoCalc as a project-centered workflow rather
than a single notebook interface.

Key message:

- Start in Jupyter when exploration is the right mode.
- Move stable work into `.py` files, modules, and tests.
- Use the terminal for real Linux package and runtime work.
- Bring results into LaTeX, markdown, slides, or teaching material.
- Let Codex work across the boundaries between files, notebooks, terminals,
  packages, and papers.

The public guide is `index.html`.
